# キーボード
多くのデスクトップではコピー・ペースト等の汎用的なアプリの基本的機能を
キーボードショートカットから実行している。

ここでは本DE環境が具体的にどのようにキーバインドを制御するのかを定義する。

## コマンドによる宣言型KM(キーボードマネージャ)
本仕様ではcliによってあらかじめ宣言されたキーバインドのみを扱う。全てのキーバインドはユーザの宣言のみによって変更される。

本DEでは、キーバインドの設定を定義するためのコマンドとしてkmを用いる。

各キーバインドは名前に対応するUNIXソケットへイベントを送信する
ソケットパスは /run/de/km/<name>.sock とする

push起動、pull起動はアプリ側の依存となる。

送信されるイベントは以下の構造を持つ：

- type: "key"
- keys: キー集合
- event: "press" | "release"

### キーバインドの割り当て
指定されたキー集合に対してアプリを割り当てる
```shell
km [super] launcher
```

割り当てがないキーは[wm](./モニタ.md)に
フォーカスされたアプリに流される。

キーバインドにマッチした入力は消費され、wmおよびアプリには送信されない。
### 同時押し
発火条件に複数のキーを割り当てる
```shell
km [super,c] copy
```
キー集合は順序を持たない集合として扱われる

キー集合が成立した瞬間に press イベントを送信し、
条件を満たさなくなった瞬間に release イベントを送信する。

成立状態が継続している間、kmは追加のイベントを送信しない。

### 長押し
```shell
km [super] launcher_if_hold_cheat #長押しはアプリ側
```
kmはキーの押下（press）および離上（release）のイベントを通知する

### キーの競合
それぞれのバインドで競合するキーがある場合、
```shell
km [super] launcher
km [super,c] copy
```
最長一致で解決する。

キーバインドの選択は、キー集合が成立し、離された瞬間（release時）に行われる。

その間、そのキー集合に宣言したアプリに対してのみrelease通知を送り、
集合に属するそれぞれのキーに対しては、零集合になるまでイベントを送らない。
```txt
[super]  #super press
↓
[super,c] #super,c press
↓
[super] #super,c release
↓
[] #not send super release
```
### 状態の参照
kmコマンド単体では登録されたショートカットの状態を取得する
```shell
km
```
出力
```shell
km [super,c] copy
km [super] launcher_if_hold_cheat #長押しはアプリ側
```
