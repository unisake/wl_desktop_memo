## linuxディストリビューション
linuxディストリビューションとはlinuxカーネルに実用的なソフトウェアを乗せたものになります。linuxカーネル自体は扱いづらいため、このような形で様々なディストリビューションがあります。

具体例
- Ubuntu:初心者にも使いやすいソフトウェアを集めたディストリビューション。
- Arch:最小限のソフトウェアだけで構築するやや上級向けのディストリビューション。

など。
## パッケージ管理
各ディストリビューションでは実用的なソフトウェア群をパッケージという形で管理・配布しています。配布の形式には様々な方式があります。
- Ubuntu - aptパッケージ:安定性を重視し、検証されたバージョンが提供される。
- Arch - pacmanパッケージ:比較的新しいパッケージが提供される
## ディスプレイサーバ
linuxでGUIを扱うにはアプリの描画結果を画面に表示させる仕組みが必要になります。現代では主に2つの選択肢があります。
- X11:UNIX時代から続く伝統的なサーバ。互換性が高い。
- wayland:X11の設計の古さを見直し、改良されたサーバ。

現代ではモニタ等のGUI描画にとどまらず、マウスやキーボードといったデバイスをアプリにどう接続するかを決めています。

このようにディスプレイサーバを通してDE環境が動作するのが現代におけるlinuxDEの仕組みとなっています。
### ディスプレイサーバの役割
上記で説明した通りディスプレイサーバの目的はアプリと入出力デバイスの通信を制御する処理であることがわかります。
それぞれデバイスやアプリケーションの役割をどのように設定するかはディスプレイサーバの設計次第であり、DE環境の挙動において重要な点を占めていると言えるでしょう。
### 仕様書との乖離点
仕様書で解説した通り今回のDE実装の本懐はUI・UXというものをアプリから切り離し、
DEの仕事として割り当てることです。その目標として仕様書で実際に説明された矩形配置やキーバインド等のデバイスとアプリの通信規約を実装することが主な実務となりそうです。

## モニタ
ユーザはコマンドまたはスクリプトで作業に最適な矩形配置を宣言します。
そのため動的なウィンドウ配置は廃止し、CLIによる宣言型によってGUIを扱います。

ウィンドウ分割を定義します
```shell
wm [0,0,500,500]
```
空間全体のグラフのうち、指定された領域ノードに対する局所更新のみを行う
**クライアントを勝手に実行しない**
```shell
wm [0,0,500,500] | browser https://chatgpt.com/
wm [500,500,1000,1000] | code ./test.c
```
パイプでつなげると指定した領域にアプリが起動されます。
### 使用例：jsonから取り込む
```json
{
  "sarfaceA": [0,0,500,500],
}
```

```py
import json
import subprocess

cfg = json.load(open("config.json"))
x, y, w, h = cfg["sarfaceA"]

cmd = f'wm [{x},{y},{w},{h}] | browser https://chatgpt.com/'
subprocess.run(cmd, shell=True)
```
