# 仕様

本稿ではwaylandを想定としたDE環境の定義を行う。

具体的にはディスプレイサーバに送られる入力・出力イベントを基に、
多くのPCで動作するモニタ・キーボード・マウスについての仕様をまとめる。

## 全体仕様

すべてのデバイス・アプリ間の挙動をCLIによるテキストベースの通信で行うこととします。

基本操作

```shell
DEコマンド args [実行するコマンド]
```

本DEは高レベルなUI要素（タブ等）を提供せず、
それらはユーザ空間のアプリケーションによって構築されることを前提とする。

## デバイス

devコマンドによって役割が重複する複数デバイスに対して制御を行う

### デバイス確認
`dev`は引数なしで実行された場合、
現在有効化されているデバイス（active）の一覧を表示する。
```shell
dev
```
出力領域が宣言されていないプロセスへの紐づけ

GUIプロセスがwmバインドを持たずに起動された場合、

browser &

システムのデフォルト動作に従いレイアウトが決定される。

デフォルト動作はフルスクリーンであり、設定ファイル~/.conf/desktopによって変更可能である。

default_layout = fullscreen
```
monitor monitorA 1920x1080
monitor monitorB 2560x1440
keyboard keyboardA layout=jp
pointer mouseA dpi=800
```
---
`-a`オプションを指定した場合、
利用可能なすべてのデバイス（active および inactive）を表示する。
```shell
dev -a
```
出力
```
monitor monitorA 1920x1080 active
monitor monitorB 2560x1440 inactive
keyboard keyboardA layout=jp active
keyboard keyboardB layout=us inactive
pointer mouseA dpi=800 active
pointer mouseB dpi=1000 inactive
```

### 接続
```shell
dev -c monitorB
dev -c keyboardA
```
指定したデバイスを有効化し、論理空間に追加する。
### 切断
```shell
dev -d monitorB
dev -d keyboardA
```
指定したデバイスを無効化し、論理空間から除外する。

### 各種デバイスへの設定
各種デバイスの設定は専用コマンドで行うこととする

- `wm` [モニタ管理](./モニタ.md)を担当する
- `km` [キーボード管理](./キーボード.md)を担当する
- `mm` [マウス管理](./マウス.md)を担当する
