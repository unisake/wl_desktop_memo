# モニタ

多くのデスクトップアプリケーションはウィンドウという矩形を操作して、
モニタへ映るアプリの表示範囲を制御している。

ここでは本DE環境が具体的にどのように矩形操作するのかを定義する。

## コマンドによる宣言型WM(ウィンドマネージャ)

本仕様では、ユーザの意図しない自動配置（動的レイアウト変更）は行わない。
すべての配置はユーザの宣言によってのみ変更される。

また、全体仕様の通り、コマンドによる宣言によってGUIを扱う。

本DEでは、ウィンドウ配置を定義するためのコマンドとして wm を用いる。

### 領域へのアプリプロセスの紐づけ
```shell
#wm [x,y,w,h]
wm [0,0,500,500] browser https://chatgpt.com/
wm [500,500,1000,1000] code ./test.c
```
- x, y は矩形の左上座標を表す
- width, height は矩形の幅および高さを表す
- 座標系はモニタ左上を (0,0) とする論理座標系を用いる

矩形領域の定義に続けてコマンドを記述した場合、
当該領域においてそのコマンドが実行される。

このとき wm はプロセスの起動と配置の両方を宣言的に定義する。

### 領域が宣言されていないプロセスへの紐づけ
GUIプロセスがwmバインドを持たずに起動された場合、
```shell
browser &
```
システムのデフォルト動作に従いレイアウトが決定される。

デフォルト動作はフルスクリーンであり、
設定ファイル`~/.conf/desktop`によって変更可能である。
```txt
default_layout = fullscreen
```

### 状態の取得

`wm`コマンドを単体で実行すると現在のウィンドウ状態が取得できる。

```shell
wm
```
出力形式は以下とする：`<rect> <command...> pid=<pid>`
```
[0,0,500,500] browser https://chatgpt.com/ pid=1234
[500,500,1000,1000] code ./test.c pid=5678
```
出力はZ順に従って下位から上位の順に並ぶ。

### 領域の移動

同じプロセスのwmコマンドを複数回宣言した場合、
```shell
wm [0,0,500,500] browser https://chatgpt.com/
wm [500,500,1000,1000] browser https://chatgpt.com/
```
当該プロセスは最後に宣言された矩形領域にのみバインドされる。
それ以前に定義された領域とのバインドは解除される。
**プロセスの複製等は行わない。**

### Z順

複数のプロセスが同一または重複する矩形領域にバインドされた場合、
それらはZ順を持つ。
```shell
wm [0,0,500,500] browser https://chatgpt.com/
wm [0,0,500,500] browser code ./test.c
```
Z順は宣言順によって決定され、
後に宣言されたプロセスほど前面に配置される。

最前面に配置されたプロセスはフォーカスを持ち、
キーボードおよびポインタ入力を受け取る。

それ以外のプロセスはバックグラウンドとして扱われ、
入力を受け取らないが、実行は継続される。

## まとめ

`wm`はレイアウト状態の定義および適用のみを責務とし、
プロセスの生成・終了・管理は対象外とする。

## 仕様例

### 例1：jsonから取り込む
```json
{
  "surfaceA": [0, 0, 500, 500]
}
```
wmはあくまで宣言であることが確認できる
```py
import json
import subprocess

cfg = json.load(open("config.json"))
x, y, w, h = cfg["surfaceA"]

cmd = f'wm [{x},{y},{w},{h}] browser https://chatgpt.com/'
subprocess.run(cmd.split())
```
### 例2：Node.js等のツールによるタブ機能実装
下図の通り、矩形領域A,Bを定義します．
![](../img/二つの領域.png)
写真10：二つの領域

領域Bにアプリを接続する。
```bash
wm $領域B 何かのアプリ
```

領域Aに対してwebview等で以下のjsが実行されるGUIアプリケーションを作成します。
```js
//提供：ChatGPT
document.getElementById("openApp").addEventListener("click", () => {
  fetch("/run", {
    method: "POST",
    body: JSON.stringify({
      cmd: ["wm", "$領域B", "別のアプリ"]
    })
  });
});
```
このときの領域を図示すると、
```bash
wm $領域Ａ 切り替えアプリ
```
